## 20260516毕业设计

### 毕设简介

基于STM32F103C8T6的定时巡逻安防小车。3.3V的18650锂电池×2供电，AMS1117-5.0V和AMS1117-3.3V降压模块降压后给各个硬件供电，陶片电容和铝电解电容给电源电路和驱动电路消峰和稳压。使用L298N模块驱动电机运动。OLED显示屏显示信息，运动状态、位置区段、前方距离、有无人员。两个按键，作为手动输入信号。具备定时启动功能。通过反射式红外传感器感受黑色线条；通过超声波传感器感受距离信息；通过一侧的反射式红外传感器识别标志黑点，计算小车处于的大致区段；通过热释电传感器识别人体信号；通过LED、蜂鸣器、ISD语音播报模块进行本地报警；通过ESP-01S与巴法云物联网平台（bemfa.com）建立TCP连接，采用平台自定义的轻量文本协议实现发布/订阅通信。小车通过串口AT指令向平台发送报警和位置信息，平台将消息推送到手机APP；手机APP也可下发控制指令（如前进、后退等），ESP-01S接收后经串口转发给STM32解析执行。

### 初始化

1.LED：PB14，高电平点亮。
2.蜂鸣器：PB15，高电平鸣叫。
3.ISD：PA8，高电平时播放录音，低电平时停止，即触发信号是高电平，而非上升沿。
4.循迹：从左到右：PB5（最左）、PB6（左中）、PB7（中间）、PB8（右中）、PB9（最右）,权重分配为-2,-1,0,1,2。传感器在地面上方时感应到返回信号，输出低电平，在黑线上时感应不到信号，输出高电平。代码中 track_pins[] 顺序应与上述物理顺序一致，weights[] = {-2, -1, 0, 1, 2}。
5.超声波：PB10、PB11，使用TIM2。首先PB10置高电平至少10us，过一会儿PB11收到持续一段时间的高电平，通过定时器获取高电平的持续时间，通过公式s=t×340/2，计算传感器距前方的距离。两次触发信号之间最小间隔200ms，可通过维护一个变量记录上次测距的时间戳，每次进入主循环检查时间差，不足 200ms 则跳过本次测距，主程序应保证该点，以避免回声干扰。
6.人体感应：PB12感应小车后方、PB13感应小车前方，当收到高电平时代表识别到人体。每次触发后，输出信号由高变低，会有2.5s的封锁时间，期间不响应人员有无情况。
7.电机驱动：PA6、PA7是左两轮、右两轮的EN使能口，通过输出不同的PWM分别控制左右电机的转速，PA4PA5控制左边两个车轮的正反转，PB0PB1控制右边两个车轮的正反转。PA4高电平、PA5低电平正转，反之反转；PB0和PB1控制转向同理；都为高电平（11）或都为低电平（00）时都是制动；EN为0时无论转向情况均为自由停止（几乎用不上）。
8.定位：PA1，收到高电平时代表感应到黑色标志点。防抖计次，与标志点总数取余，输出字符串“小车在x与x+1之间”。
9.通信：PA9PA10，通过USART与ESP-01S通信，进而依照MQTT协议与物联网云平台通信。因为中文支持解析有难度，上报下发的信息使用英文表达。通信协议：ESP-01S与巴法云服务器建立TCP连接（AT+CIPSTART="TCP","bemfa.com",8344），之后通过自定义文本指令实现订阅和发布。
订阅主题：发送 cmd=1&uid=641dba44ecd7004fce71e73028c876a1&topic=car。
发布消息：发送 cmd=2&uid=641dba44ecd7004fce71e73028c876a1&topic=car&msg=消息内容。
心跳维持：每60秒内发送 cmd=0&msg=ping。
STM32代码中需将上述指令拼接成字符串，通过USART1发送给ESP-01S。注意：指令中的双引号无需转义，只需正常的C字符串转义（如 \r\n）。
10.OLED使用的现成的程序和库，PA2-SCK，PA3-SDA。
11.按键：PA11和PA12，按下后会输入低电平。上电后选择运动模式，一个代表循迹运动、一个避障运动；再选择启动模式，一个立即启动、一个定时启动；定时启动需要设置时间，一个增加小时数、一个增加分钟数。
12.测距用的TIM2，驱动电机产生使能PWM的TIM3，ESP用的USART1。

### 主程序流程

上电，进入待机状态。按键选择运动模式，再选择立即启动/定时启动，定时启动可通过按键设置小时、分钟数，采用相对延时，涉及RTC和停止模式。
当APP下发运动指令时，自动切换遥控运动状态，可遥控运动。
循迹运动模式下，通过读取5路红外传感器，加权平均计算偏差，PID输出转向、PWM，特殊情况下全为高电平/低电平时制动停车。同时超声波实时测距、人体红外检测，障碍物过近或有人员闯入时，停车报警“前方存在障碍物”/“异常人员闯入”。
避障运动模式下，默认直行，当前方距离小于设定值时停车，向右后方倒车，再尝试直行。只进行人体红外检测，识别到人员时，停车报警“异常人员闯入”，不需要“前方存在障碍物”的警报，因为本模式的运动转向就是根据障碍物情况的。
遥控运动状态下，也是只进行人体红外检测。
报警分为本地和云端，本地为LED点亮、蜂鸣器鸣叫、ISD播放录音；云端为向上发送警情和位置信息，进而发送到手机APP。报警后持续检测，当传感器恢复正常状态（障碍物消失/人员离开）时，自动退出报警状态并恢复巡逻，关闭本地报警、更新云端数据，关闭LED、蜂鸣器、ISD，上发位置信息和“No person”/“No obstacle”消息。

### 附注

1.系统建议使用状态机，待机状态（停止模式，可用RTC定时唤醒或者手动按键唤醒）、遥控运动状态（运动指令见下方物联网云平台服务指令）、循迹巡逻状态、避障巡逻状态、报警状态（制动停车+本地云端报警）。
2.涉及存储的部分尝试使用环形存储区，比如通信的接收。
3.低功耗部分,使用 RTC 闹钟 + Stop 模式，闹钟触发后唤醒执行巡逻.
4.采用巴法云平台，UID和主题需在平台创建，具体用法见初始化部分。UID为“641dba44ecd7004fce71e73028c876a1”，主题是“car”。
5.本示例项目仅为参考，请按照效果最佳的方案编程。**即可使用我未曾提到的方法。**
