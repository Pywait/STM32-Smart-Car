1. 按键，GPIO上拉输入PA2(\PA3)复位为按键按下，添加防抖代码。按键按下即启动主程序（主程序见最后）。
2. 运动模块，使用A6A7的TIM3计时器功能生成PWM信号，A4A5A6控制小车左侧两轮，A7B0B1控制小车右侧两个轮子。使用了H桥驱动模块，00、11停止，01正转，10反转。
3. 循迹模块：B5、B6、B7、B8、B9，为GPIO，上拉输入，当收到数字信号为低电平时代表在黑色预定轨迹上。5个端口所控制的传感器模块在小车前方的从左向右1、2、5、4、3位置。尝试使用PID算法实现循迹功能。中断，AFIO、EXTI、NVIC。*轮询是标准做法*
4. 超声波模块：USART3的B10为TX、B11为RX，通过输出一个大于10us的高电平，会收到一个带时间信息的高电平。通过TIM2计时器，数值＋溢出次数获得高电平的保持时间，通过公式“距离=时间✖342.63/2”算出距离，当距离小于10cm时代表**有障碍物**，报警；停止运动。中断，TIM2、AFIO、EXTI、NVIC。*必须用中断*
5. 人体感应模块：端口B12负责感知小车后方，B13负责小车前方。使用GPIO-下拉输入，当收到高电平且保持0.5s，则代表有**异常人员闯入**，报警；停止运动。中断，AFIO、EXTI、NVIC。*轮询就够*
6. 位置信息：PA1，GPIO上拉输入，低电平时加一。创建一个数组，通过累加接收到的次数取余数，推算出小车的大致位置。
7. ESP8266模块：A11复位时通信模块复位；A12连接通信模块的GPIO0悬空为工作模式、下拉为下载模式；通讯使用的USART1，A9为TX、A10为RX。波特率：；停止位：；校验位：；。

- 本地报警功能：B14置位点亮LED灯、B15置位蜂鸣器发声、A8**保持置位**播放录音。

## 20260513毕业设计

### 毕设简介
基于STM32F103C8T6的定时巡逻安防小车。3.3V的18650锂电池×2供电，AMS1117-5.0V和AMS1117-3.3V降压模块降压后给各个硬件供电，陶片电容和铝电解电容给电源电路和驱动电路消峰和稳压。使用L298N模块驱动电机运动。OLED显示屏显示信息，运动状态、位置区段、有无人员。两个按键，作为手动输入信号。通过反射式红外传感器感受黑色线条；通过超声波传感器感受距离信息；通过一侧的反射式红外传感器识别标志黑点，计算小车处于的大致区段；通过热释电传感器识别人体信号；通过LED、蜂鸣器、ISD语音播报模块进行本地报警；通过ESP-01S与物联网云平台按照MQTT协议通信，发送警情、位置，云平台下发给APP响应的信号。APP可向云端发送控制信号，比如前进后退，数据通过ESP和串口解析后小车按命令执行动作。通讯数据采用JSON格式。具备定时启动功能。

### 初始化
1.LED：PB14，高电平点亮。
2.蜂鸣器：PB15，高电平鸣叫。
3.ISD：PA8，高电平时播放录音，低电平时可停止。
4.循迹：从左到右：PB5（最左）、PB6（左中）、PB9（中间）、PB8（右中）、PB7（最右）,权重分配为-2,-1,0,1,2，在地面上时感应到返回信号，输出低电平，在黑线上时感应不到信号，输出高电平。
5.超声波：PB10、PB11，使用TIM2。首先PB10置高电平至少10us，过一会儿PB11收到持续一段时间的高电平，通过定时器获取高电平的持续时间，通过公式s=t×340/2，计算传感器距前方的距离。两次触发信号之间最小间隔200ms。
6.人体感应：PB12感应小车后方、PB13感应小车前方，当收到高电平时代表识别到人体。每次触发后，输出信号由高变低，会有2.5s的封锁时间，期间不响应人员有无情况。
7.电机驱动：PA6、PA7是EN使能口，通过输出不同的PWM分别控制左右电机的转速，PA4PA5控制左边两个车轮的正反转，PB0PB1控制右边两个车轮的正反转。01正转，10反转，11和00都是制动，EN为0时为自由停止（几乎用不上）。
8.定位：PA1，收到高电平时代表感应到黑色标志点。计次，与标志点总数取余，输出字符串“小车在x与x+1之间”。
9.通信：PA9PA10，通过USART与ESP-01S通信，进而依照MQTT协议与物联网云平台通信。语法为AT指令，在Keil程序中，C语言需要转义斜杠、双引号、逗号，而AT指令在发送的信息部分，双引号内需要用转义符号对双引号和逗号进行转义，即转义两遍。ESP的网络连接已经配好，需要MQTT连接，先串口输出“AT+MQTTUSERCFG=0,1,"Car2","0G3y0rg843","version=2018-10-31&res=products%2F0G3y0rg843%2Fdevices%2FCar2&et=1779692492&method=md5&sign=YAUHO4QfJK2gwL9xtDQkCQ%3D%3D",0,0,"设置 MQTT 用户属性，再“AT+MQTTCONN=0,"mqtts.heclouds.com",1883,1”连接 MQTT Broker。物联网云平台本毕设用到的话题类型可见最后的物模型。
10.OLED使用的现成的程序和库，PA2-SCK，PA3-SDA。
11.按键：PA11和PA12，按下后会输入低电平。
12.测距用的TIM2，驱动电机产生使能PWM的TIM3，ESP用的USART1。

### 主程序流程
上电，进入待机状态。按键选择运动模式，再选择立即启动/定时启动，定时启动可通过按键设置小时、分钟数，采用相对延时，涉及RTC和停止模式。
当APP下发运动指令时，自动切换遥控运动状态，可遥控运动。
循迹运动模式下，通过读取5路红外传感器，加权平均计算偏差，PID输出转向、PWM，特殊情况下全为高电平/低电平时制动停车。同时超声波实时测距、人体红外检测，障碍物过近或有人员闯入时，停车报警“前方存在障碍物”/“异常人员闯入”。
避障运动模式下，默认直行，当前方距离小于设定值时停车，向右后方倒车，再尝试直行。只进行人体红外检测，识别到人员时，停车报警“异常人员闯入”。
遥控运动状态下，也是只进行人体红外检测。
报警分为本地和云端，本地为LED点亮、蜂鸣器鸣叫、ISD播放录音；云端为向上发送警情和位置信息，进而发送到手机APP。报警后持续检测，当传感器恢复正常状态（障碍物消失/人员离开）时，自动退出报警状态并恢复巡逻，关闭本地报警、更新云端数据。
