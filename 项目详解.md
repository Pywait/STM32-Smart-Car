# STM32 定时巡逻安防小车 — 项目详解

---

## 一、项目概览

这是一个基于 **STM32F103C8T6**（Cortex-M3）的毕业设计项目。小车上装备了多种传感器和执行器，能在无人值守的情况下自动巡逻：

- **循迹巡逻**：沿着地面的黑色引导线自动行驶
- **避障巡逻**：超声波实时测距，遇障自动绕行
- **人体检测**：前后两个热释电传感器，发现人员闯入立即报警
- **云端通信**：通过 ESP-01S 经 MQTT 连接 OneNET 云平台
- **手机遥控**：APP 下发前进/后退/左转/右转/停止指令
- **定时启动**：RTC 闹钟 + STOP 模式实现低功耗定时巡逻
- **本地报警**：LED 闪烁、蜂鸣器鸣叫、ISD 语音播报

整个程序基于 **状态机** 架构，主循环周期 ~10ms，所有传感器非阻塞运行。

## 四、代码逐行详解

### 4.1 `Hardware/hardware.h` — 引脚映射

这个头文件是本项目的"接线图"。所有硬件引脚都在这里集中定义，方便修改和查阅。

```c
#define LED_PORT GPIOB           // LED → GPIOB
#define LED_PIN GPIO_Pin_14      // 引脚 14
```
- `LED_PORT` / `LED_PIN` — 决定 LED 挂在哪个 GPIO 组的哪个引脚。PB14 高电平点亮。
- 类似的有 `ISD_PORT PA8`（语音模块）、`KEY_PORT PA11/PA12`（按键）、`BUZZER_PORT PB15`。

```c
#define ULTRASONIC_TX_PIN GPIO_Pin_10   // PB10 → Trig
#define ULTRASONIC_RX_PIN GPIO_Pin_11   // PB11 → Echo
```
- 超声波 Trig 和 Echo 引脚。Echo 同时用作 EXTI11 中断输入。

```c
#define PIR_FRONT_PIN GPIO_Pin_13       // PB13 前方
#define PIR_BACK_PIN  GPIO_Pin_12       // PB12 后方
```
- 前后两个 PIR 传感器，分别感测前方和后方的人员闯入。

```c
// PB5(最左) PB6(左中) PB9(中间) PB8(右中) PB7(最右)
// 权重：-2, -1, 0, 1, 2
```
- 5 路循迹传感器的物理布局。注意 PB9 是中间传感器（不是 PB7），这行注释就是接线图。

```c
#define MOTOR_L_PWM_A_PIN GPIO_Pin_6   // PA6 TIM3 CH1 → 左电机 PWM
#define MOTOR_R_PWM_A_PIN GPIO_Pin_7   // PA7 TIM3 CH2 → 右电机 PWM
#define MOTOR_L1_A_PIN GPIO_Pin_4      // PA4 → 左电机方向 1
#define MOTOR_L2_A_PIN GPIO_Pin_5      // PA5 → 左电机方向 2
#define MOTOR_R1_B_PIN GPIO_Pin_0      // PB0 → 右电机方向 1
#define MOTOR_R2_B_PIN GPIO_Pin_1      // PB1 → 右电机方向 2
```
- L298N 驱动接口：PWM 控制速度，方向引脚控制正反转。

### 4.2 `System/Delay.c` — 延时与 tick 时钟

```c
static volatile uint32_t tick_count = 0;
```
- `volatile` 告诉编译器这个变量可能被中断修改，防止优化。tick_count 每毫秒递增一次，作为整个系统的时间基准。

```c
uint32_t Delay_GetTick(void) { return tick_count; }
```
- 返回当前毫秒数。主程序和传感器都用它做非阻塞的时间判断。

```c
void Delay_us(uint32_t xus) {
    SysTick->LOAD = 72 * xus;     // 72MHz, 1us = 72 个时钟周期
    SysTick->VAL = 0x00;          // 清空当前值
    SysTick->CTRL = 0x00000005;   // 使能 SysTick，时钟源 = HCLK/1
    while(!(SysTick->CTRL & 0x00010000)); // 等待 COUNTFLAG 置位
    SysTick->CTRL = 0x00000004;   // 关闭 SysTick（保留时钟源选择）
}
```
- 微秒级阻塞延时。通过 SysTick 的 LOAD 寄存器设置延时长度（72 个时钟 = 1us），然后轮询 COUNTFLAG。
- `0x00000005` 二进制是 `...0101`：bit0=使能，bit2=HCLK/1。

```c
void Delay_ms(uint32_t xms) {
    while(xms--) {
        Delay_us(1000);
        tick_count++;      // 每毫秒递增 tick
    }
}
```
- 毫秒级延时。每 1ms 递增 `tick_count`。阻塞延时期间 tick 依然在走，其他用 `Delay_GetTick()` 判断超时的代码能正常工作。

### 4.3 `Hardware/LED.c` — LED 控制

```c
void LED_Init(void) {
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB, ENABLE); // 开启 GPIOB 时钟
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_PP;      // 推挽输出
    GPIO_InitStructure.GPIO_Pin = LED_PIN;                 // PB14
    GPIO_Init(LED_PORT, &GPIO_InitStructure);
    GPIO_ResetBits(LED_PORT, LED_PIN);    // 初始化熄灭
}
```
- STM32 外设使用前必须先开启时钟。`RCC_APB2PeriphClockCmd` 打开 GPIOB 的时钟。
- `GPIO_Mode_Out_PP` 推挽输出 — 可以输出高电平和低电平。

```c
void LED_ON(void)  { GPIO_SetBits(LED_PORT, LED_PIN); }   // 高电平点亮
void LED_OFF(void) { GPIO_ResetBits(LED_PORT, LED_PIN); } // 低电平熄灭
```
- `SetBits` 将引脚输出 1，`ResetBits` 输出 0。

```c
void LED_Turn(void) {  // 翻转
    if (GPIO_ReadOutputDataBit(LED_PORT, LED_PIN) == 0)
        GPIO_SetBits(LED_PORT, LED_PIN);
    else
        GPIO_ResetBits(LED_PORT, LED_PIN);
}
```

### 4.5 `Hardware/ISD.c` — 语音播报模块

```c
static uint8_t ISD_Playing = 0;  // 保存播放状态
```
- 静态变量记录当前是否在播放。

```c
void ISD_PLAYL_ON(void) {
    if (ISD_Playing == 0) {
        ISD_Playing = 1;
        GPIO_SetBits(ISD_PORT, ISD_PIN);     // 上升沿开始播放
    }
    else if (ISD_Playing == 1) {
        GPIO_ResetBits(ISD_PORT, ISD_PIN);   // 先拉低
        Delay_us(10);                         // 保持 10us
        GPIO_SetBits(ISD_PORT, ISD_PIN);     // 再拉高 → 重新触发上升沿
    }
}
```
- ISD1820 语音模块是上升沿触发播放。如果正在播放，需要先拉低再拉高产生一个新的上升沿来重新触发。
- `Delay_us(10)` 保证低电平持续时间足够让模块识别到边沿跳变。

```c
void ISD_PLAYL_OFF(void) {
    GPIO_ResetBits(ISD_PORT, ISD_PIN);
    ISD_Playing = 0;
}
```

### 4.6 `Hardware/Key.c` — 按键扫描（阻塞式）

```c
void Key_Init(void) {
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPU;  // 上拉输入
    GPIO_InitStructure.GPIO_Pin = KEY_PIN1 | KEY_PIN2;  // PA11, PA12
    GPIO_Init(KEY_PORT, &GPIO_InitStructure);
}
```
- `GPIO_Mode_IPU` 内部上拉输入。按键另一端接 GND，按下时引脚读到低电平。
- 上拉输入在浮空时保持高电平，按下拉低，省去外部上拉电阻。

```c
uint8_t Key_GetNum(void) {
    uint8_t KeyNum = 0;
    if (GPIO_ReadInputDataBit(KEY_PORT, KEY_PIN1) == 0) {  // 检测到按下
        Delay_ms(20);                   // 消抖：等待 20ms
        if (GPIO_ReadInputDataBit(KEY_PORT, KEY_PIN1) == 0) {  // 再次确认
            while (GPIO_ReadInputDataBit(KEY_PORT, KEY_PIN1) == 0);  // 等待松手
            Delay_ms(20);               // 松手消抖
            KeyNum = 1;
        }
    }
    // KEY2 同理
    return KeyNum;
}
```
- **消抖机制**：机械按键按下瞬间会抖动（几毫秒内多次跳变），通过两次检测 + 延时来避开抖动区间。
- **松手检测**：`while(...==0)` 阻塞等待按键释放，确保一次按下只返回一次键值，不会连续触发。
- 这个函数是阻塞的 — 如果用户一直按着不放，程序会卡在 `while` 循环里。

### 4.7 `Hardware/Ultrasonic.c` — 超声波非阻塞测距

这是最复杂的传感器模块之一。核心思路是：**不在中断里等回响，而是靠状态机在 main loop 里轮询**。

```c
volatile uint8_t Ultrasonic_Flag = 0;       // 0=空闲/完成, 1=等待下降沿
volatile uint16_t Ultrasonic_Timer = 0;      // TIM2 溢出次数（每溢出 = 1ms）
volatile uint32_t Ultrasonic_Distance = 0;   // 缓存的上次测距结果（mm）
```
- 三个变量都在中断和 main loop 之间共享，所以用 `volatile` 防止编译器优化。

```c
static uint32_t last_trig_time = 0;   // 上次触发时间戳
static uint8_t trig_pending = 0;       // 0=空闲, 1=已触发等待回响
```

**初始化：**
```c
void Ultrasonic_Init(void) {
    // GPIO 初始化：PB10 推挽输出(Trig), PB11 下拉输入(Echo)
    // TIM2 配置：72MHz / 72 = 1MHz, 自动重装 1000 → 1ms 溢出一次
    // EXTI11 配置：PB11 上升沿+下降沿触发中断
    // NVIC：TIM2 和 EXTI15_10 中断优先级配置
}
```
关键参数：
- TIM2 预分频 72 → 1MHz，1000 周期 → 1ms 中断一次
- 即 `Ultrasonic_Timer` 每加 1 代表 1ms

**更新状态机（主循环每次调用）：**
```c
void Ultrasonic_Update(void) {
    uint32_t now = Delay_GetTick();
    if (trig_pending) {
        if (Ultrasonic_Flag == 0)        // 下降沿已到，测距完成
            trig_pending = 0;
        else if (now - last_trig_time > 30) {  // 30ms 超时
            TIM_Cmd(TIM2, DISABLE);
            Ultrasonic_Flag = 0;
            Ultrasonic_Distance = 0;      // 无回响 → 距离 0
            trig_pending = 0;
        }
    }
    if (!trig_pending && (now - last_trig_time >= 200)) {  // 200ms 间隔
        // 发送 15us 高电平到 Trig 引脚
        GPIO_SetBits(ULTRASONIC_PORT, ULTRASONIC_TX_PIN);
        Delay_us(15);
        GPIO_ResetBits(ULTRASONIC_PORT, ULTRASONIC_TX_PIN);
        last_trig_time = now;
        trig_pending = 1;
    }
}
```
- **200ms 间隔限制**：HC-SR04 数据手册要求两次触发间隔至少 60ms，这里设 200ms 保证余量。
- **30ms 超时**：声音在空气中的传播速度约 343m/s，30ms 对应距离约 5 米（343 × 0.03 / 2 ≈ 5.1m），超过 HC-SR04 最大量程。
- **非阻塞**：没有 `while` 等待，每次调用只检查状态、做少量操作就返回。

**中断处理（EXTI15_10_IRQHandler）：**
```c
void EXTI15_10_IRQHandler(void) {
    if (EXTI_GetITStatus(EXTI_Line11) != RESET) {
        if (Ultrasonic_Flag == 0) {       // 上升沿 → 开始计时
            Ultrasonic_Flag = 1;
            Ultrasonic_Timer = 0;
            TIM_SetCounter(TIM2, 0);
            TIM_Cmd(TIM2, ENABLE);        // 启动 TIM2 计时
        }
        else if (Ultrasonic_Flag == 1) {  // 下降沿 → 停止计时，计算距离
            TIM_Cmd(TIM2, DISABLE);
            // 距离 = 时间 × 声速 / 2 = (Timer × 1000 + CNT) × 343 / 2000 (mm)
            Ultrasonic_Distance = (Ultrasonic_Timer * 1000 + TIM_GetCounter(TIM2)) * 343 / 2000;
            Ultrasonic_Flag = 0;
        }
        EXTI_ClearITPendingBit(EXTI_Line11);
    }
}
```
- 上升沿触发：TIM_Timer 和 TIM_GetCounter 一起组成高精度计时（1us 分辨率）。
- 下降沿触发：距离 mm = (总时间 us) × 343 (m/s) / 2000。除以 2000 是因为 mm = m × 1000，声速往返所以要 /2。
- 具体：`超声波行走距离(mm) = 时间(us) × 343 (mm/ms) / 1000 / 2`，简化后就是 `时间 × 343 / 2000`。

**TIM2 中断（1ms 溢出一次）：**
```c
void TIM2_IRQHandler(void) {
    if (TIM_GetITStatus(TIM2, TIM_IT_Update) != RESET) {
        Ultrasonic_Timer++;              // 每 1ms 加 1
        TIM_ClearITPendingBit(TIM2, TIM_IT_Update);
    }
}
```

### 4.8 `Hardware/PIR.c` — 人体热释电检测

```c
#define PIR_LOCKOUT_MS 2500   // 2.5s 封锁时间
```
- HC-SR501 人体感应模块触发后，输出高电平持续一段时间后变低，然后有 2.5s 的封锁期（不响应任何检测）。代码里用软件锁定期模拟这个行为。

```c
uint8_t PIR_examine(void) {
    static uint32_t last_trig_time = 0;   // 上次触发时间戳
    uint32_t now = Delay_GetTick();

    if (now - last_trig_time < PIR_LOCKOUT_MS)
        return 0;              // 封锁期内不响应

    uint8_t PIR_examine_Num = 0;
    if (GPIO_ReadInputDataBit(PIR_PORT, PIR_FRONT_PIN) == 1) {
        Delay_us(10);          // 消抖
        if (GPIO_ReadInputDataBit(PIR_PORT, PIR_FRONT_PIN) == 1) {
            PIR_examine_Num = 1;   // 前方有人
            last_trig_time = now;  // 开始封锁
        }
    }
    // 后方同理，返回 2
    return PIR_examine_Num;
}
```
- 返回值：0=无人，1=前方有人，2=后方有人。
- `last_trig_time` 是静态局部变量，函数退出后值不丢失。

### 4.9 `Hardware/PWM.c` — 电机 PWM

```c
void PWM_Init(void) {
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM3, ENABLE);  // TIM3 → APB1 时钟
    
    // PA6, PA7 复用推挽输出（TIM3 CH1, CH2）
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AF_PP;
    GPIO_InitStructure.GPIO_Pin = MOTOR_L_PWM_A_PIN | MOTOR_R_PWM_A_PIN;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    TIM_TimeBaseInitStructure.TIM_Period = 100 - 1;       // ARR=99
    TIM_TimeBaseInitStructure.TIM_Prescaler = 36 - 1;     // PSC=35
    // 72MHz / 36 = 2MHz, 2MHz / 100 = 20kHz PWM 频率
```
- PWM 频率 = 72MHz / 36 / 100 = **20kHz**。20kHz 听不到人耳噪音，电机响应也够快。
- 占空比范围 0~99，对应 0%~100%。
- `TIM_OCMode_PWM1`：CNT < CCR 时输出有效电平，CNT >= CCR 时输出无效电平。

### 4.10 `Hardware/Motor.c` — L298N 电机驱动

**编码规则**（L298N 逻辑真值表）：
- `01` = 正转
- `10` = 反转
- `11` 或 `00` = 制动

```c
void Motor_L_Setspeed(int8_t speed) {
    if (speed > 0) {          // 正转：PA4=1, PA5=0
        GPIO_SetBits(GPIOA, MOTOR_L1_A_PIN);
        GPIO_ResetBits(GPIOA, MOTOR_L2_A_PIN);
        PWM_SetCompare1(speed);           // 速度 = speed%
    }
    else if (speed < 0) {     // 反转：PA4=0, PA5=1
        GPIO_ResetBits(GPIOA, MOTOR_L1_A_PIN);
        GPIO_SetBits(GPIOA, MOTOR_L2_A_PIN);
        PWM_SetCompare1(-speed);          // 取绝对值
    }
    else {                    // 制动：PA4=1, PA5=1
        GPIO_SetBits(GPIOA, MOTOR_L1_A_PIN | MOTOR_L2_A_PIN);
        PWM_SetCompare1(0);               // 速度 0
    }
}
```
- 正值 = 正转，负值 = 反转，0 = 制动。
- 注意 `PWM_SetCompare1(-speed)` 中的负号：因为 speed 为负，取绝对值后写入 CCR。

```c
void Motor_Brake(void) {
    GPIO_SetBits(GPIOA, MOTOR_L1_A_PIN | MOTOR_L2_A_PIN);
    GPIO_SetBits(GPIOB, MOTOR_R1_B_PIN | MOTOR_R2_B_PIN);
    PWM_SetCompare1(100);
    PWM_SetCompare2(100);
}
```

### 4.12 `Hardware/Track.c` — 5 路循迹 + PID 控制

**传感器布局与权重：**

```c
static const uint16_t track_pins[5] = {
    TRACK_L1_PIN, TRACK_L2_PIN, TRACK_M_PIN, TRACK_R2_PIN, TRACK_R1_PIN
};
static const int8_t weights[5] = {-2, -1, 0, 1, 2};  // 左负右正
```
- 物理顺序：最左(L1) → 左中(L2) → 中间(M) → 右中(R2) → 最右(R1)
- 权重对称分布：偏离中心线越远权重绝对值越大。

**PID 参数：**
```c
static float Kp = 12.0f;   // 比例系数
static float Ki = 0.5f;    // 积分系数
static float Kd = 2.0f;    // 微分系数
static float integral = 0.0f;
static float last_error = 0.0f;
static float max_integral = 100.0f;  // 积分限幅
```

**传感器编码：**
```c
uint8_t Track_GetSensorCode(void) {
    uint8_t SensorCode = 0;
    // L1 → bit4, L2 → bit3, M → bit2, R2 → bit1, R1 → bit0
    if (GPIO_ReadInputDataBit(TRACK_PORT, TRACK_L1_PIN) == BLACK_LEVEL)
        SensorCode |= (1 << 4);
    // ... 以此类推
    return SensorCode;
}
```
- 返回 5 位编码，如 `0b00100` 表示只有中间传感器检测到黑线。

**偏差计算：**
```c
float Track_GetError(void) {
    uint8_t sensor_code = Track_GetSensorCode();
    int16_t sum_weight = 0;
    uint8_t active_count = 0;
    for (uint8_t i = 0; i < 5; i++) {
        if (sensor_code & (1 << (4 - i))) {
            sum_weight += weights[i];
            active_count++;
        }
    }
    if (active_count == 0) {
        // 全部悬空（脱线）：沿用上次偏差方向继续找线
        if (last_error > 0) return 2;
        else if (last_error < 0) return -2;
        else return 0;
    }
    return (float)sum_weight / active_count;  // 加权平均
}
```
- 本质是加权平均重心计算。例如 L1 和 M 检测到：`(-2 + 0) / 2 = -1`，表示偏左 1 个单位。
- 当所有传感器都没检测到黑线（脱线），利用 `last_error` 保持上次转向方向继续寻找。

**PID 输出：**
```c
float Track_PIDControl(float error) {
    float p_term = Kp * error;             // 比例：即时响应
    integral += error;                     // 积分：消除静差
    if (integral > max_integral) integral = max_integral;  // 积分限幅
    if (integral < -max_integral) integral = -max_integral;
    float i_term = Ki * integral;          // 积分项
    float d_term = Kd * (error - last_error);  // 微分：预测趋势
    last_error = error;
    return p_term + i_term + d_term;       // 总输出
}
```
- **比例项**：当前偏差越大，转向越猛。
- **积分项**：长期累积偏差（如小车持续偏左）逐渐加大修正量。
- **微分项**：偏差变化率，抑制过冲，让转向更平滑。

**PID 在 TrackingMode 中的应用：**
```c
float error = Track_GetError();
float pid_output = Track_PIDControl(error);
int16_t left_speed = BASE_SPEED - pid_output;   // 左轮减速
int16_t right_speed = BASE_SPEED + pid_output;  // 右轮加速
// 限幅 0~99
```
- 偏差负（偏左）→ pid_output 负 → 左轮加速、右轮减速 → 向右修正。

**全黑/全白制动：**
```c
if (sensor_code == 0x00 || sensor_code == 0x1F) {
    Car_Brake();   // 脱离轨道或到达终点 → 制动停车
}
```
- `0x00` = 全白背景（脱线），`0x1F` = 全黑（终点或 T 型路口）。

### 4.13 `Hardware/Position.c` — 红外定位

```c
uint16_t Position_Get(uint16_t TPNum) {
    if (lastState == 0 && GPIO_ReadInputDataBit(...) == 1) {  // 检测到上升沿
        Delay_us(20);
        if (GPIO_ReadInputDataBit(...) == 1) {  // 再次确认
            Position0++;         // 标志点计数
            lastState = 1;
        }
    }
    if (lastState == 1 && GPIO_ReadInputDataBit(...) == 0) {  // 下降沿
        // 更新状态
        lastState = 0;
    }
    Position = Position0 % TPNum;   // 取余，获得当前区段
    return Position;
}
```
- PA1 上检测反射式红外传感器，识别地面上黑色标志点。
- 上升沿计数一次，下降沿恢复等待。
- 返回值 `Position0 % TPNum` 表示当前在第几个区段（例如总 10 个标志点，返回 0~9）。

**边缘检测技巧**：用 `lastState` 记录上一次电平状态，只有上升沿时才计数。这种"边沿触发"方式比电平方式更可靠，不会因传感器持续输出高电平而重复计数。

### 4.14 `Hardware/ESP.c` — ESP-01S MQTT 通信

这是项目最复杂的部分，涉及 AT 指令、MQTT 协议、环形缓冲区和 JSON 格式。

**环形缓冲区：**
```c
#define RX_BUF_SIZE 256
static volatile uint8_t rx_buffer[RX_BUF_SIZE];
static volatile uint16_t rx_head = 0;   // 写指针
static volatile uint16_t rx_tail = 0;   // 读指针
```
- 先进先出（FIFO）：USART1 中断收到数据写入 `rx_head` 位置，主循环从 `rx_tail` 读取。
- `(rx_head + 1) % RX_BUF_SIZE == rx_tail` 时表示缓冲区满，新数据丢弃。

**上行发送：**
```c
void ESP_SendAT(const char* cmd) {
    ESP_SendString((uint8_t*)cmd);
    ESP_SendString((uint8_t*)"\r\n");    // AT 指令以 \r\n 结尾
}
```
所有的 MQTT 操作都是通过 AT 指令完成的。

**MQTT 初始化序列（三步）：**
```c
void ESP_MQTTInit(void) {
    Delay_ms(1000);               // 等 ESP-01S 启动
    // 1) AT+MQTTUSERCFG=...     设置 MQTT 客户端身份
    // 2) AT+MQTTCONN=...        连接到 OneNET MQTT Broker
    // 3) AT+MQTTSUB=...         订阅 RemoteCtrl 和 PatrolMode 两个服务话题
}
```
- 每个指令后 `ESP_WaitResp("OK", timeout)` 等待 ESP 返回 OK 确认，带超时防止卡死。

**数据上报 — 双重转义：**
```c
void ESP_SendAlarm(uint8_t type) {
    // 人员报警：
    ESP_SendAT("AT+MQTTPUB=0,\"$sys/0G3y0rg843/Car2/thing/event/post\","
        "\"{\\\"id\\\":\\\"124\\\"\\,\\\"params\\\":{\\\"AlarmPerson\\\""
        ":{\\\"value\\\":{\\\"info\\\":\\\"异常人员闯入\\\"}}}}\",0,0");
}
```
- **双重转义原因**：
  1. **C 语言层面**：`\"` 表示双引号字符，`\\` 表示反斜杠字符
  2. **AT 指令层面**：ESP-AT 固件要求 JSON 字符串内的双引号和逗号用 `\"` 和 `\,` 转义
- 最终 ESP-01S 发送到网络的原始 JSON：
  ```json
  {"id":"124","params":{"AlarmPerson":{"value":{"info":"异常人员闯入"}}}}
  ```

**下行命令解析：**
```c
static void ParseMQTTLine(const char* line) {
    // 搜索 "cmd" → 遥控命令 F/B/L/R/S
    p = strstr(line, "\"cmd\"");
    if (p) {
        current_state = STATE_REMOTE_CONTROL;  // 切换到遥控状态
        if      (strstr(p, "\"F\"")) { /* 前进 */ }
        else if (strstr(p, "\"B\"")) { /* 后退 */ }
        // ... L=左转, R=右转, S=停止
    }
    // 搜索 "mode" → 巡逻模式 0/1/2
    p = strstr(line, "\"mode\"");
    if (p) {
        if      (strstr(p, "\"mode\":0")) { current_mode = MODE_TRACKING; }
        else if (strstr(p, "\"mode\":1")) { current_mode = MODE_OBSTACLE_AVOIDANCE; }
        else if (strstr(p, "\"mode\":2")) { current_state = STATE_REMOTE_CONTROL; }
    }
}
```
- APP 下发 JSON 命令 → OneNET → ESP-01S → USART1 中断 → 环形缓冲区 → 主循环 `ESP_ProcessCommands()` 拼行 → `ParseMQTTLine()` 解析
- 只做简单字符串匹配，不引入 JSON 解析库，节省 Flash。

**行拼装（ESP_ProcessCommands）：**
```c
void ESP_ProcessCommands(void) {
    while (rx_buffer_get(&byte)) {
        if (byte == '\n') {
            line_buf[line_pos] = '\0';       // 字符串结束
            if (line_pos > 0) ParseMQTTLine(line_buf);  // 解析
            line_pos = 0;
        } else if (byte != '\r') {
            if (line_pos < LINE_BUF_SIZE - 1)
                line_buf[line_pos++] = (char)byte;  // 拼到行缓冲区
        }
    }
}
```
- `\r\n` 是 AT 指令的行结束符，这里忽略 `\r`，遇到 `\n` 表示一行完整数据到达。
- 行缓冲区 128 字节，超过的部分丢弃（防止恶意数据撑爆内存）。

#### StandbyMode() — 待机 STOP 模式（低功耗）

```c
static void StandbyMode(void) {
    // 显示 "STANDBY"
    // 停止电机、关 LED、关蜂鸣器
```

**第一步：保存超声波的 EXTI11 配置**
```c
    uint32_t save_exticr2 = AFIO->EXTICR[2];  // 引脚映射
    uint32_t save_imr = EXTI->IMR;            // 中断屏蔽
    uint32_t save_emr = EXTI->EMR;            // 事件屏蔽
    uint32_t save_rtsr = EXTI->RTSR;          // 上升沿触发
    uint32_t save_ftsr = EXTI->FTSR;          // 下降沿触发
```
- 超声波用 PB11 → EXTI11 中断模式检测 Echo。
- 待机时要把 EXTI11 改造成 PA11（按键）→ 事件模式唤醒 STOP，所以要先保存超声波的配置。

**第二步：配置按键唤醒**
```c
    GPIO_EXTILineConfig(GPIO_PortSourceGPIOA, GPIO_PinSource11);  // EXTI11 → PA11
    EXTI->IMR  &= ~EXTI_Line11;      // 关中断模式
    EXTI->EMR  |=  EXTI_Line11;      // 开事件模式（事件唤醒不进 ISR）
    EXTI->RTSR &= ~EXTI_Line11;      // 关上升沿
    EXTI->FTSR |=  EXTI_Line11;      // 开下降沿（按下为低电平）
    EXTI->PR    =  EXTI_Line11;      // 清挂起位
```
- **事件模式 vs 中断模式**：事件模式可以唤醒 STOP 但不触发 ISR，避免了与超声波 EXTI15_10_IRQHandler 冲突。
- KEY2 (PA12 → EXTI12) 同样配置。

**第三步：进入 STOP**
```c
    PWR_EnterSTOPMode(PWR_Regulator_LowPower, PWR_STOPEntry_WFI);
    // ========== 按键唤醒（从这继续执行） ==========
    SystemInit();   // 恢复 HSE + PLL 时钟（STOP 模式关闭了时钟）
```
- `PWR_EnterSTOPMode`：CPU 进入停止模式，所有时钟停止，SRAM 内容保持。
- 唤醒后第一件事：`SystemInit()` 重新初始化系统时钟到 72MHz。

**第四步：等待按键释放**
```c
    {
        uint32_t t = Delay_GetTick();
        while (GPIO_ReadInputDataBit(KEY_PORT, KEY_PIN1) == 0) {
            if (Delay_GetTick() - t > 500) break;
        }
        Delay_ms(20);
    }
```
- 防止唤醒的按键按下事件被 main loop 的 `Key_GetNum()` 再次捕获，造成状态跳跃。

**第五步：恢复超声波配置**
```c
    AFIO->EXTICR[2] = save_exticr2;  // 恢复引脚映射
    EXTI->IMR = save_imr;
    EXTI->EMR = save_emr;
    EXTI->RTSR = save_rtsr;
    EXTI->FTSR = save_ftsr;
```

#### TrackingMode() — 循迹巡逻

```c
    distance = Ultrasonic_GetDistance();    // 读取最近一次超声结果
    pir_state = PIR_examine();              // 人体检测
```

### 4.16 `User/main.c` — 主函数与状态转移

**全局变量（定时启动参数）：**
```c
static uint16_t timed_hours = 0;      // 定时小时

```
- 这些变量声明为 `static`（文件作用域），在本文件内有效。
- 注意注释："会被 STOP 模式下 SRAM 保持" — 因为 STOP 模式不丢失 SRAM 内容，所以定时参数不会丢失。

**RTC 定时启动（RTC_EnterTimedStartup）：**
```c
static void RTC_EnterTimedStartup(uint16_t minutes) {
    // 1) 使能 PWR/BKP 时钟
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_PWR | RCC_APB1Periph_BKP, ENABLE);
    PWR_BackupAccessCmd(ENABLE);     // 允许访问备份域

    // 2) 配置 LSI 为 RTC 时钟源（LSI ≈ 40kHz）
    RCC_LSICmd(ENABLE);
    while (RCC_GetFlagStatus(RCC_FLAG_LSIRDY) == RESET);  // 等待 LSI 稳定
    RCC_RTCCLKConfig(RCC_RTCCLKSource_LSI);
    RCC_RTCCLKCmd(ENABLE);
    RTC_WaitForSynchro();

    // 3) 40kHz / 40000 = 1Hz
    RTC_SetPrescaler(40000 - 1);
    RTC_WaitForLastTask();

    // 4) 计数器从 0 开始，闹钟 = minutes × 60
    RTC_SetCounter(0);
    RTC_WaitForLastTask();
    RTC_SetAlarm(minutes * 60);
    RTC_WaitForLastTask();

    // 5) 配置 EXTI Line17 为 RTC 闹钟唤醒
    // 6) 显示 "TIMED START" 后进入 STOP
    PWR_EnterSTOPMode(PWR_Regulator_LowPower, PWR_STOPEntry_WFI);

    // 唤醒后：SystemInit() + 清除闹钟标志
    // 继续执行巡逻
}
```
- **LSI**（低速内部振荡器）≈ 40kHz，在 STOP 模式下仍然运行，驱动 RTC 计数。
- 预分频 40000 → RTC 计数器频率 1Hz（1 秒加 1）。
- 闹钟值 `minutes × 60`：例如 5 分钟 = 300，5 分钟后 RTC 计数器到达 300 触发闹钟中断。
- EXTI Line17 是 RTC 闹钟的唤醒线，可以将 MCU 从 STOP 模式唤醒。

**主循环 — 按键驱动的状态转移：**
```c
while (1) {
    KeyNum = Key_GetNum();       // 获取按键（阻塞，等松手）

    switch (current_state) {
    case STATE_STANDBY:
        if (KeyNum == 1) current_state = STATE_MODE_SELECT;
        break;

    case STATE_MODE_SELECT:
        if (KeyNum == 1) { current_mode = MODE_TRACKING; ... }   // KEY1=循迹
        else if (KeyNum == 2) { current_mode = MODE_OBSTACLE_AVOIDANCE; ... }  // KEY2=避障
        break;

    case STATE_STARTUP_SELECT:
        if (KeyNum == 1) 立即启动;
        else if (KeyNum == 2) 进入定时设置 STATE_TIMED_SETUP;
        break;

    case STATE_TIMED_SETUP:
        // KEY1 切换调小时/分钟
        // KEY2 +1
        // 3 秒无按键自动确认 → RTC_EnterTimedStartup()
        break;
    }

    StateMachine_Update();           // 执行当前状态的处理函数
    Ultrasonic_Update();             // 非阻塞超声测距状态机
    ESP_ProcessCommands();           // 处理 ESP 下行数据

    Delay_ms(10);                    // 主循环周期 ≈10ms
}
```
- 按键驱动状态转移（`KeyNum` 判断），状态机内部驱动具体行为（`StateMachine_Update`）。
- `Key_GetNum()` 返回后，根据当前状态决定 KEY1/KEY2 的作用。
- 主循环周期约 10ms，各传感器状态机按需工作（超声 200ms 触发一次，PIR 随时可读，循迹 50ms 控制一次）。

### 4.17 `User/stm32f10x_it.c` — 中断服务

```c
void RTC_IRQHandler(void) {
    if (RTC_GetITStatus(RTC_IT_ALR) != RESET) {
        RTC_ClearITPendingBit(RTC_IT_ALR);   // 清除 RTC 闹钟中断标志
        RTC_WaitForLastTask();
    }
}

void RTCAlarm_IRQHandler(void) {
    if (EXTI_GetITStatus(EXTI_Line17) != RESET) {
        EXTI_ClearITPendingBit(EXTI_Line17);  // 清除 EXTI17 中断标志
    }
}
```
- `RTC_IRQHandler` 清除 RTC 模块内部的闹钟标志。
- `RTCAlarm_IRQHandler` 清除 EXTI 线的挂起位。两者都是"清理 + 不做其他事"，因为唤醒后 main 函数继续执行，不需要中断里做额外工作。

## 六、项目亮点总结

1. **非阻塞架构**：所有传感器（超声波、PIR、ESP 通信）都是非阻塞的，主循环保持 ~10ms 稳定周期
2. **状态机设计**：9 个状态清晰分离，每个状态独立处理，状态转移集中管理
3. **低功耗策略**：待机 STOP 模式功耗 μA 级别，RTC 定时唤醒实现无人值守巡逻
4. **双重转义通信**：C 语言 → AT 指令 → MQTT JSON，三层嵌套转义正确实现
5. **环形缓冲区**：256 字节 FIFO 高效处理不确定长度的 ESP 异步串口数据
6. **PID 循迹控制**：加权平均偏差 + PID 修正，转向平滑，脱线可自动找回
7. **自动恢复**：报警状态检测传感器恢复正常后自动退出报警、恢复巡逻
